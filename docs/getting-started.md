# Getting Started with Wizly

## 1. Installing the Extension

Install Wizly from the [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/):

1. Open VS Code
2. Go to the **Extensions** tab (`Ctrl+Shift+X`)
3. Search for **Wizly**
4. Click **Install**

Wizly is immediately available in your workspace after installation.

---

## 2. Transforming a File

Open an HTML file generated by Magic xpa Web Client and run a transformation:

1. Open the file in the editor
2. Press `Ctrl+Shift+P` (Windows/Linux) or `Cmd+Shift+P` (macOS) to open the command palette
3. Type **Wizly** and select `Wizly: Transform Current File`

Wizly applies all active rules and formats the file using Prettier.

> **Tip:** Need to process multiple files at once? Use `Wizly: Transform All Uncommitted Files` to transform all files that have changed relative to Git.

---

## 3. Exporting and Configuring Settings

Wizly settings can be configured through VS Code settings, or you can export a project file (`.vswizly.js`) to share settings with your team.

### Exporting Settings

1. Open the command palette with `Ctrl+Shift+P`
2. Select `Wizly: Export Settings`
3. A `.vswizly/` folder is created in the root of your workspace containing the config file

### Enabling smartLabelMatcher

A useful setting to turn on is **smartLabelMatcher**. This feature automatically moves standalone `<label>` elements into the corresponding `<mat-form-field>` as a `<mat-label>`.

```js
module.exports = {
  smartLabelMatcher: {
    enabled: true,
    labelPrefix: "L_",
    controlPrefix: ["V_", "P_"]
  }
};
```

See the [README](../README.md#%EF%B8%8F-configuration) for a full overview of all available settings.

---

## 4. Exporting and Customizing Templates

Wizly uses EJS templates to generate HTML structures. You can export the built-in templates and customize them per project.

1. Open the command palette with `Ctrl+Shift+P`
2. Select `Wizly: Export Templates`
3. The templates are copied into the `.vswizly/templates/` folder in your workspace

You can then edit the `.ejs` files as needed. Wizly automatically picks up your local versions — any template present in the project takes precedence over the built-in version.

See [docs/templates.md](templates.md) for an overview of all templates and available variables.

---

## 5. Exporting Rules (Advanced)

> **Note:** Only do this if you know what you are doing. The built-in rules cover most use cases, and modifying them can cause unexpected behavior if you are not familiar with how the regex processing works.

If you do want to customize the rules:

1. Open the command palette with `Ctrl+Shift+P`
2. Select `Wizly: Export Advanced Rules`
3. The built-in rules are copied to your workspace as a starting point

You can then add, modify, or disable rules as needed. Rules are executed top to bottom — order matters.

See [docs/rules.md](rules.md) for a detailed explanation of all rule fields, regex flags, named groups, and tips.
